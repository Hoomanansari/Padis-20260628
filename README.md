# Padis-20260628

## معرفی پروژه

این Repository برای مدیریت مستندات پروژه تهیه پروپوزال سیستم هوشمند و یکپارچه کنترل تردد، کنترل دسترسی، مدیریت امنیت و مدیریت پارکینگ یک مجموعه آموزشی / مدرسه بزرگ ایجاد شده است.

پروپوزال توسط شرکت **HLWL** به عنوان نماینده خاورمیانه ارائه می‌شود و راهکارها و تجهیزات پیشنهادی بر پایه محصولات و فناوری‌های برند **Syncolony** خواهند بود.

نسخه اولیه اسناد به زبان فارسی تهیه می‌شود و پس از تکمیل و تأیید محتوا، نسخه نهایی حرفه‌ای به زبان انگلیسی تولید خواهد شد.

---

## هدف Repository

هدف این Repository ایجاد یک مرجع ساختاریافته، قابل نسخه‌بندی و قابل استفاده توسط ابزارهای AI برای تولید و تکمیل اسناد پروژه است.

این Repository باید کمک کند که:

- نیازمندی‌های کارفرما به‌صورت شفاف ثبت شوند.
- فرضیات و ابهامات از تصمیمات قطعی جدا شوند.
- اسناد Proposal به‌صورت مرحله‌ای و قابل کنترل تکمیل شوند.
- منابع پروژه، پلان‌ها و دیتاشیت‌ها متمرکز نگهداری شوند.
- نسخه‌های Word و PDF از روی محتوای نهایی Markdown تولید شوند.
- پروژه در آینده برای پروژه‌های مشابه قابل استفاده مجدد باشد.

## اصل کلیدی پروژه

اصل طراحی راهکار پیشنهادی این است:

> هر فرد یا خودرو باید فقط در زمان مجاز، مکان مجاز و با سطح دسترسی مجاز امکان تردد داشته باشد.

بنابراین پروژه فقط یک سیستم باز و بسته کردن درب نیست، بلکه یک راهکار جامع برای کنترل سیاست‌های تردد، مانیتورینگ، هشدار، گزارش‌گیری، مدیریت مهمانان، مدیریت خودروها و یکپارچه‌سازی با سامانه‌های مدرسه است.

## محدوده کلی پروژه

محدوده پروژه شامل طراحی مفهومی و پیشنهادی برای موارد زیر است:

- کنترل دسترسی درب‌ها و فضاهای مدرسه
- کنترل ورود و خروج دانش‌آموزان
- کنترل ورود و خروج پرسنل، معلمان و مدیران
- مدیریت والدین، مهمانان و مراجعان
- مدیریت پیمانکاران و سرویس مدارس
- کنترل دسترسی کلاس‌ها، آزمایشگاه‌ها و فضاهای حساس
- کنترل ورود به طبقات و نواحی غیرمجاز
- مدیریت پارکینگ و خودروهای مجاز
- پشتیبانی از ANPR، RFID و UHF برای خودروها
- استفاده از گیت‌های کنترل تردد
- استفاده از تشخیص چهره و RFID برای احراز هویت
- مانیتورینگ، هشدار و گزارش‌گیری
- امکان اتصال به CCTV، اعلام حریق، API مدرسه و سایر سامانه‌ها
- بررسی تجهیزات بازرسی مانند فلزیاب، X-Ray و کنترل موبایل به عنوان Option

## نکات مهم برای کار با AI

هنگام استفاده از ChatGPT، Codex، Cursor یا سایر ابزارهای AI روی این Repository، باید این اصول رعایت شود:

1. ابتدا `PROJECT_STATUS.md` خوانده شود.
2. تصمیمات قطعی از `DECISIONS.md` استخراج شود.
3. ابهامات از `Open_Questions.md` بررسی شود.
4. فرضیات از `Assumptions.md` جدا از تصمیمات قطعی در نظر گرفته شود.
5. تغییرات جدید باید با ساختار Repository هماهنگ باشد.
6. اسناد Proposal نباید بدون هماهنگی با نیازمندی‌ها و سناریوها نوشته شوند.
7. هر تغییر مهم باید در `TODO.md` یا `DECISIONS.md` منعکس شود.

# معماری کلان سیستم پیشنهادی

## 1. هدف معماری
معماری پیشنهادی برای این پروژه با هدف ایجاد یک سیستم یکپارچه، قابل توسعه و قابل مدیریت برای کنترل تردد، کنترل دسترسی، مدیریت امنیت و پارکینگ مدرسه طراحی می‌شود.

هدف اصلی سیستم این است که هر فرد یا خودرو فقط در زمان مجاز، مکان مجاز و با سطح دسترسی مجاز امکان تردد داشته باشد.

## 2. لایه‌های معماری پیشنهادی

### 2.1 لایه کاربران و اشخاص
این لایه شامل گروه‌های زیر است:

- دانش‌آموزان
- معلمان
- پرسنل اداری
- مدیران
- والدین
- مهمانان
- پیمانکاران
- سرویس مدارس
- نیروهای امنیتی
- اپراتورها و مدیران سیستم

### 2.2 لایه تجهیزات میدانی
این لایه شامل تجهیزات نصب‌شده در سایت است:

- گیت‌های کنترل تردد
- دستگاه‌های تشخیص چهره
- کارت‌خوان‌ها
- قفل‌های الکترونیکی
- کنترلرهای درب
- دوربین‌های تشخیص چهره یا CCTV
- Barrier Gate پارکینگ
- ANPR Camera
- RFID Reader
- UHF Reader و Tag
- تجهیزات بازرسی در صورت نیاز

### 2.3 لایه نرم‌افزار مرکزی
نرم‌افزار مرکزی وظیفه مدیریت کاربران، نقش‌ها، سیاست‌های دسترسی، رخدادها، گزارش‌ها و یکپارچه‌سازی را بر عهده دارد.

قابلیت‌های کلیدی:

- تعریف گروه، نقش و سطح دسترسی
- تعریف سیاست دسترسی بر اساس زمان، مکان و نقش
- مانیتورینگ زنده
- گزارش‌گیری
- مدیریت رخدادها و هشدارها
- مدیریت پارکینگ
- مدیریت مهمانان
- مدیریت نقشه و جانمایی تجهیزات
- API / Web Service Integration

### 2.4 لایه یکپارچه‌سازی
این لایه برای ارتباط با سامانه‌های دیگر مدرسه و زیرساخت‌های امنیتی استفاده می‌شود:

- سامانه دانش‌آموزی
- سامانه منابع انسانی یا پرسنلی
- Active Directory در صورت وجود
- سیستم CCTV
- سیستم اعلام و اطفاء حریق
- سامانه اطلاع‌رسانی به والدین
- سامانه‌های گزارش‌گیری مدیریتی

## 3. جریان کلی داده
1. اطلاعات کاربران از نرم‌افزار یا سامانه مدرسه ثبت یا دریافت می‌شود.
2. گروه‌ها، نقش‌ها و سیاست‌های دسترسی تعریف می‌شوند.
3. اطلاعات کاربران روی تجهیزات مرتبط Sync می‌شود.
4. کاربر در نقطه دسترسی شناسایی می‌شود.
5. نرم‌افزار یا کنترلر مجوز را بر اساس سیاست‌ها بررسی می‌کند.
6. در صورت تأیید، درب، گیت یا Barrier باز می‌شود.
7. رخداد ثبت و برای گزارش‌گیری، مانیتورینگ یا هشدار استفاده می‌شود.

## 4. معماری پیشنهادی برای مدرسه
با توجه به پلان‌های دریافت‌شده، معماری باید حداقل نواحی زیر را پوشش دهد:

- ورودی اصلی سایت
- Guard Room و Reception
- Drop Area
- بخش KG
- بخش پسران
- بخش دختران
- کلاس‌ها
- آزمایشگاه‌ها
- راهروهای اصلی
- Administration
- Library
- Gym
- Clinic
- IT Room
- پارکینگ معلمان
- خروجی‌های دانش‌آموزان
